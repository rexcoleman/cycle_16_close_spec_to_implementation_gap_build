# Design Decision: Two-Domain framing for governance

We adopt the Two-Domains framing (Strategy vs Operations) as the conceptual lens for
all governance discussion. This is a design decision and a documentation record; it
binds no running code. The rationale is that separating WHAT/WHY from HOW/WHEN reduces
operational-punt rationalizations. Nothing here executes; this records the chosen
mental model per DP#26 (citation-based activation, needs no implementation).
